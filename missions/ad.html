Contexte

Objectif

Mettre en place un Active Directory fonctionnel avec une structure organisationnelle (OU), des utilisateurs, des groupes, puis appliquer des stratégies de groupe (GPO) pour centraliser la gestion des postes et des droits utilisateurs.

Création — Active Directory

1

Installation du rôle AD DS

Sur Windows Server, installation du rôle Active Directory Domain Services (AD DS) via le Gestionnaire de serveur, puis promotion du serveur en contrôleur de domaine.

Gestionnaire de serveur → Ajouter des rôles → AD DS
Promotion : Créer une nouvelle forêt → Nom de domaine : gc-cicar.es
Niveau fonctionnel : Windows Server 2019 → Définir le mot de passe DSRM
2

Création des Unités d'Organisation (OU)

Depuis Utilisateurs et ordinateurs Active Directory (ADUC), création d'une arborescence d'OU reflétant l'organisation.

Clic droit sur le domaine → Nouveau → Unité d'organisation
Création des OU : Resp. Agence, Employés, Groupes, Ordinateurs
3

Création des utilisateurs et groupes

Création des comptes utilisateurs dans leur OU respective, puis constitution des groupes de sécurité pour faciliter la gestion des droits d'accès.

Utilisateur : Nom, prénom, identifiant (prenom.nom), mot de passe temporaire + changement obligatoire à la 1ère connexion
Groupes : type Sécurité, portée Globale — ajout des utilisateurs dans leur groupe correspondant
4

Jonction des machines au domaine

Depuis chaque poste client Windows, jonction au domaine gc-cicar.es pour centraliser l'authentification et permettre l'application des GPO.

Système → Paramètres avancés → Nom de l'ordinateur → Modifier → Domaine : gc-cicar.es
Authentification avec un compte administrateur du domaine
Redémarrage obligatoire — le poste apparaît ensuite dans l'OU :OU_Ordinateurs

Création — Stratégies de Groupe (GPO)

5

Création et liaison des GPO

Depuis la Console de gestion des stratégies de groupe (GPMC), création des GPO et liaison à l'OU ciblée.

Gestion des stratégies de groupe → Clic droit sur une OU → Créer un objet GPO et le lier ici
Nommage explicite : Interdiction_personnalisation, Deploiement_Partage_Employes…
Modification via l'Éditeur de gestion des stratégies de groupe
6

GPO appliquées

Principales stratégies de groupe mises en place dans le cadre de cette mission :

Nom GPOOU cibleParamètre configuré

Interdiction_personnalisationOU_UtilisateursInterdiction de personnaliser le fond d'écran.
Deploiement_Partage_EmployesOU_EmployesDeploiement du lecteurs réseaux.
Wallpaper_EmployesOU_EmployesDéfinition du fond d'écarn pour les employés.
7

Vérification et application des GPO

Après modification des GPO, vérification de leur bonne application sur les postes clients.

Forcer la mise à jour : gpupdate /force sur le poste client
Vérifier les GPO appliquées : gpresult /r

Gestion des droits et permissions

8

Partages réseau et permissions NTFS

Création de dossiers partagés sur le serveur de fichiers avec des permissions définies par groupes de sécurité AD.

Partage réseau : \\SRV2K19-GC\Partage_Employes — accessible uniquement par GG_Employes
Permissions NTFS : Lecture seule (GDL_Partage_Employes_R), Modification (GDL_Partage_Employes_RW), Contrôle total (admins)
Bonne pratique : toujours gérer les droits via les groupes AD(et locaux de préfèrence), jamais directement sur les comptes utilisateurs
